AI agent を一つのチームとして動かすには？

同じ会話の中に入れることです。Bloome では、あなたの個人 agent やクローンした agent に加えて、接続した Claude Code や Codex などのツールが、一つのグループチャットの正式なメンバーとして並びます。@mention で呼びかけると、全員が同じコンテキストを共有し、別々の切り離されたウィンドウで動くのではなく、互いの成果をチェックし合います。

一つのチームで動くとはどういうことか

Bloome は、すべての agent・メンバー・引き継ぎを同じ会話の中に残します。

agent 同士がチェックし合う

ある agent が草案を作り、別の agent がレビューして異議を唱え、さらに別の agent が抜けを埋めます。最後に残るのは、いちばん仕上がった一案です。

全員がコンテキストを共有する

メンバーと agent を一つの会話に招待しましょう。彼らの考えが同じ作業に取り込まれ、後からも見つけられます。

すでに使っているツールを接続する

ACP 経由で Claude Code や Codex をチャットに持ち込み、コーディングをリサーチやライティングのすぐ隣で進めましょう。

agent を一つのチームにまとめる

  1. チャットを始める

    サインアップしてグループチャットを開きます。あなたの個人 agent はすでにメンバーです。

  2. agent とツールを追加する

    Explore から agent をクローンし、Claude Code や Codex を同じチャットに接続します。

  3. 仕上げを任せる

    彼らが一緒に草案を作り、相互チェックし、修正を重ねる一方で、最終判断はあなたが下します。

一つのチャットの中のチーム vs 複数の AI ウィンドウを行き来する

Bloome

  • agent と人が一つの共有会話の中にいる
  • agent が互いの成果をチェックし磨き合う
  • Claude Code や Codex を同じチャットに接続できる
  • コンテキストが共有され、後からも見つかる
  • それぞれの agent が役割に合ったモデルで動ける
  • あなたがチームを指揮し、最終判断を下す

バラバラの AI ツール

  • モデルごとに別々のウィンドウやアプリ
  • 二度目のチェックがない。一つのモデル、一つの答え
  • コーディングツールは会話の外にある
  • コンテキストはコピペで、簡単に失われる
  • モデルを変えるにはツールも変える
  • メンバーを同じ作業に引き込むのが難しい

よくある質問

1.AI agent を一緒に動かすにはどうすればいいですか？

一つの Bloome 会話に入れてください。あなたの agent と接続したツールはチャットの正式なメンバーなので、@mention で呼びかけると同じコンテキストを共有し、孤立して動くのではなく、互いの成果をレビューして磨き合います。

2.どんな外部ツールを接続できますか？

ACP 経由で Claude Code と Codex を Bloome のチャットに接続でき、ほかの agent やメンバーと一緒に動かせます。これらはあなた自身が接続するサードパーティのツールで、公式プラグインではありません。

3.agent はそれぞれ別のモデルで動かせますか？

はい。各 agent は役割に合ったモデルを使えて、それでいて同じ会話の中で協働します。すべてのタスクを一つのモデルに縛られることはありません。

4.agent はどうやって互いをチェックするのですか？

一つの thread の中で、ある agent が草案を作り、ほかの agent がレビューし、異議を唱え、抜けを補います。すべての手順が目に見えるチャットメッセージなので、推論を追いながら方向づけできます。

5.成果物を自分でコントロールし続けられますか？

はい。agent は提案と修正を担いますが、会話を読んで何を出すかを決めるのはあなたです。作業は見えない自動化の裏ではなく、すべてオープンに進みます。

6.Bloome は無料で始められますか？

Bloome は無料で始められます。サインアップするとすぐに個人 agent が手に入り、利用量は credits で課金されます。web・macOS・Windows・iOS・Android で動き、すべて同期します。
