Tracking 1 · Consent-abhängiges Skript-Laden .bs-track-consent data-theme="dark"

Grundlagen · Analytics & Tracking

Kein Tracking-Skript lädt vor dem Consent

Das eine Muster, das alles trägt: Skripte liegen als type="text/plain" inert im HTML. So lädt der Browser sie garantiert nicht — erst die Zustimmung macht sie scharf.

1 · Skript inert einbetten

<!-- Bleibt inert: type="text/plain" führt der
     Browser nie aus. Aktiviert wird erst nach
     Zustimmung zur Kategorie "statistik". -->
<script type="text/plain" data-consent="statistik">
  // Beispiel: Analytics-Loader
  var s = document.createElement("script");
  s.src = "/js/analytics.js";
  s.defer = true;
  document.head.appendChild(s);
</script>

2 · Nach Zustimmung aktivieren

// Inerte Blöcke als echte <script>-Elemente
// einsetzen — aufrufen, sobald der Cookie-Banner
// eine Entscheidung gespeichert hat.
function aktiviereConsentSkripte(consent) {
  document.querySelectorAll('script[type="text/plain"][data-consent]')
    .forEach(function (tpl) {
      if (!consent[tpl.dataset.consent]) return;
      if (tpl.dataset.active === "true") return; // nie doppelt
      var live = document.createElement("script");
      live.textContent = tpl.textContent;
      tpl.dataset.active = "true";
      tpl.parentNode.insertBefore(live, tpl.nextSibling);
    });
}

// Beim Seitenstart mit gespeichertem Consent
// (Schlüssel wie im Cookie-Banner-Baustein):
aktiviereConsentSkripte(
  JSON.parse(localStorage.getItem("bs-consent") || "{}")
);
🍪

Das Gegenstück — Banner, Kategorien-Modal und Speicherung als JSON in localStorage — steht fertig im Baustein Utility 1 · Cookie-Consent-Banner. Dessen Skript ruft genau diese Aktivierung nach jeder Entscheidung auf.

Tracking 3 · CTA-Event-Tracking .bs-track-events [data-track] data-theme="dark"

Events

Ein Listener für alle CTAs

Kein onclick an jedem Button: ein delegierter Listener liest [data-track]-Attribute und pusht sauber in den dataLayer.

Zentraler Klick-Listener (einmal einbinden)

// dataLayer-agnostisch: GTM, Matomo Tag Manager & Co.
// lesen dasselbe Array. Der Tag Manager selbst lädt
// consent-abhängig (siehe Tracking 1).
window.dataLayer = window.dataLayer || [];

document.addEventListener("click", function (e) {
  var el = e.target.closest("[data-track]");
  if (!el) return;
  window.dataLayer.push({
    event: el.dataset.track,
    label: el.dataset.trackLabel || el.textContent.trim()
  });
});

Markup am Button

<a href="/registrieren"
   class="bs-btn bs-btn-primary"
   data-track="cta_click"
   data-track-label="Hero · Kostenlos starten">
  Kostenlos starten
</a>

<!-- ohne data-track-label wird der
     Button-Text als Label benutzt -->
<button type="button" data-track="demo_start">
  Live-Demo ansehen
</button>
Ein Ort für die LogikNeue CTAs brauchen nur ein Attribut — kein zusätzliches JS, kein Vergessen.
Konsistente Namenevent als Maschinen-Name (snake_case), label als lesbarer Kontext mit Position.
Delegation gewinntclosest() fängt auch Klicks auf Icons oder Spans im Button sauber ab.

Tracking 4 · UTM-Capture .bs-track-utm localStorage + hidden fields

Attribution

UTM-Parameter bis ins Formular tragen

Woher kam der Lead? utm_* beim Erstbesuch sichern und still in versteckte Formularfelder schreiben — der Rest des Formular-Fundaments bleibt unverändert.

Capture-Skript

// NUR nach Statistik-Consent ausführen — z. B. als
// <script type="text/plain" data-consent="statistik">
// einbetten (Muster aus Tracking 1).
(function () {
  var KEY = "bs-utm", TAGE = 30; // Speicherdauer begrenzen
  try {
    var utm = JSON.parse(localStorage.getItem(KEY) || "null");
    if (utm && Date.now() - utm.erfasst > TAGE * 864e5) {
      localStorage.removeItem(KEY); // abgelaufen → löschen
      utm = null;
    }
    if (!utm) { // Erstbesuch: Parameter aus der URL lesen
      var p = new URLSearchParams(location.search), neu = {};
      ["utm_source", "utm_medium", "utm_campaign",
       "utm_term", "utm_content"].forEach(function (k) {
        if (p.get(k)) neu[k] = p.get(k);
      });
      if (Object.keys(neu).length) {
        neu.erfasst = Date.now();
        localStorage.setItem(KEY, JSON.stringify(neu));
        utm = neu;
      }
    }
    // In versteckte Felder aller Formulare schreiben
    if (utm) document.querySelectorAll("[data-utm-field]")
      .forEach(function (f) { f.value = utm[f.dataset.utmField] || ""; });
  } catch (e) { /* Storage blockiert — Formular geht auch ohne UTM raus */ }
})();

Tracking 5 · Scroll-Tiefe & Sichtbarkeit .bs-track-scroll IntersectionObserver data-theme="dark"

Engagement

Wie weit lesen die Leute wirklich?

Scroll-Tiefe und Sichtbarkeit einzelner Sektionen — beides mit IntersectionObserver, beides garantiert nur einmal pro Seitenaufruf.

Scroll-Tiefe: 25 / 50 / 75 / 100 %

// Vier unsichtbare 1-px-Messpunkte in der Seite —
// der Observer meldet jede Marke genau einmal.
(function () {
  window.dataLayer = window.dataLayer || [];
  // Bezugsrahmen für die top:%-Werte sicherstellen
  if (getComputedStyle(document.body).position === "static") {
    document.body.style.position = "relative";
  }
  var io = new IntersectionObserver(function (entries) {
    entries.forEach(function (e) {
      if (!e.isIntersecting) return;
      io.unobserve(e.target); // einmalig
      window.dataLayer.push({
        event: "scroll_tiefe",
        label: e.target.dataset.marke + " %"
      });
    });
  });
  [25, 50, 75, 100].forEach(function (m) {
    var s = document.createElement("div");
    s.dataset.marke = m;
    s.style.cssText = "position:absolute;left:0;width:1px;" +
      "height:1px;pointer-events:none;" +
      "top:calc(" + m + "% - 2px);";
    document.body.appendChild(s);
    io.observe(s);
  });
})();

Sichtbarkeit: „Sektion gesehen“

// Markup: der Sektion nur ein Attribut geben —
// <section data-sichtbar="preise_gesehen">…</section>
(function () {
  window.dataLayer = window.dataLayer || [];
  var io = new IntersectionObserver(function (entries) {
    entries.forEach(function (e) {
      if (!e.isIntersecting) return;
      io.unobserve(e.target); // einmalig
      window.dataLayer.push({
        event: "sichtbar",
        label: e.target.dataset.sichtbar
      });
    });
  }, { threshold: 0.5 }); // halb im Viewport
  document.querySelectorAll("[data-sichtbar]")
    .forEach(function (el) { io.observe(el); });
})();

Beide Snippets laufen ohne Scroll-Listener und ohne Layout-Berechnungen im Scroll-Pfad — der Browser meldet Schnittpunkte selbst. Consent-abhängig einbetten wie in Tracking 1, Events landen im selben dataLayer wie die CTA-Klicks aus Tracking 3.

Tracking 6 · Ehrlichkeitskasten .bs-track-recht

⚖ Was ohne Consent erlaubt ist

Ohne Einwilligung bleibt dir die aggregierte, cookielose Zählung auf der Serverseite: Seitenaufrufe aus den eigenen Server-Logs oder ein selbst gehostetes Statistik-Tool, das keine Kennungen im Browser ablegt, kein Fingerprinting betreibt und einzelne Personen nicht wiedererkennt — also Antworten auf „Wie viele Aufrufe hatte /preise diese Woche?“ statt „Was hat Nutzerin X getan?“. Sobald ein Werkzeug Cookies oder localStorage fürs Tracking nutzt, Profile bildet oder Daten an Dritte überträgt, braucht es die Einwilligung vorher — genau dafür stehen die Bausteine oben.

Dieser Kasten erklärt technische Muster und ist keine Rechtsberatung — die Einordnung für den konkreten Fall gehört zu Datenschutzbeauftragten oder Kanzlei.

Tracking 7 · Checkliste .bs-track-check data-theme="dark"

Vor dem Livegang

Tracking-Checkliste

Acht Punkte, einmal durchgehen — dann ist die Seite messbar und sauber.

  1. Netzwerk-Tab leer vor Consent: Seite im Privatfenster laden — vor der Banner-Entscheidung darf kein Tracking-Request rausgehen.

  2. Banner ohne Dark Pattern: „Alle akzeptieren“ und „Nur notwendige“ gleichwertig gestaltet (Baustein Utility 1 in utility.html).

  3. Widerruf so leicht wie Zustimmung: „Cookie-Einstellungen“-Link im Footer, nach dem Downgrade location.reload().

  4. Skripte nur inert im HTML: alles Externe liegt als type="text/plain" data-consent — nie als normales <script src>.

  5. Events zentral über [data-track]: ein Listener, konsistente event-Namen in snake_case, Labels mit Position.

  6. UTM mit Ablaufdatum: Capture nur nach Consent, Speicherdauer begrenzt (30 Tage) und in der Datenschutzerklärung genannt.

  7. Datenschutzerklärung aktuell: jedes Tool mit Zweck, Anbieter, Speicherdauer und Rechtsgrundlage aufgeführt (Vorlage: datenschutz.html).

  8. Sparsam messen: nur Events erheben, die eine konkrete Entscheidung ändern — jede Kennzahl ohne Konsequenz fliegt raus.
