SEO & Meta — Snippets zum Kopieren

Acht kopierfertige Vorlagen für alles, was Suchmaschinen und Share-Vorschauen von deiner Landing Page brauchen: <head>, Social Cards, strukturierte Daten, Sprachvarianten, robots.txt und Sitemap.

Kopiert werden nur die dunklen Code-Karten. Domain lumeo.de, Marke „Lumeo" und alle Texte sind Platzhalter — vor dem Livegang durch eigene Werte ersetzen. Seitenrahmen und Label-Leisten sind Demo-Chrome: NICHT mitkopieren.

SEO 1 · Komplette <head>-Vorlage title description canonical

01

Die <head>-Vorlage

Der <head> entscheidet, wie deine Seite im Suchergebnis aussieht: Title (50–60 Zeichen, Keyword vorn, Marke hinten) und Description (120–155 Zeichen, konkreter Nutzen plus Handlungsaufforderung) sind dein Werbetext bei Google. Der Canonical-Link legt die eine gültige URL fest und verhindert Duplicate Content zwischen Varianten wie mit/ohne www.

Jede Unterseite bekommt eigenen Title, eigene Description und eigenen Canonical — nie 1:1 von der Startseite kopieren.

<!DOCTYPE html>
<html lang="de">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />

  <!-- Title: 50–60 Zeichen — Keyword vorn, Marke hinten -->
  <title>Session-Replays und Produkt-Analytics — Lumeo</title>

  <!-- Description: 120–155 Zeichen — Nutzen + Handlungsaufforderung -->
  <meta name="description" content="Sieh exakt, wie Nutzer dein Produkt bedienen: Session-Replays, Fehler-Tracking und Funnels — DSGVO-konform aus Frankfurt. Jetzt kostenlos starten." />

  <!-- Canonical: genau EINE gültige URL pro Seite (https + www konsequent) -->
  <link rel="canonical" href="https://www.lumeo.de/" />

  <!-- Färbt die Browser-Leiste auf Mobilgeräten — Markenfarbe eintragen -->
  <meta name="theme-color" content="#394dfe" />

  <!-- Favicons: SVG modern, ICO als Fallback, PNG 180×180 für iOS -->
  <link rel="icon" href="/favicon.svg" type="image/svg+xml" />
  <link rel="icon" href="/favicon.ico" sizes="32x32" />
  <link rel="apple-touch-icon" href="/apple-touch-icon.png" />

  <link rel="stylesheet" href="/css/style.css" />
  <!-- Open Graph + Twitter Cards: Abschnitt 02 · JSON-LD: Abschnitte 03–06 -->
</head>

SEO 2 · Open Graph & Twitter Cards og:image 1200×630 summary_large_image

02

Open Graph & Twitter Cards

Diese Tags bestimmen die Vorschaukarte beim Teilen in LinkedIn, WhatsApp, Slack und X. Ohne og:image wählt jede Plattform irgendein Bild von der Seite — mit einer eigenen 1200×630-Grafik behältst du die Kontrolle. Twitter/X liest die og:*-Werte als Fallback, braucht aber zwingend den Karten-Typ twitter:card.

Vorschau vor dem Livegang testen: LinkedIn Post Inspector, Meta Sharing Debugger oder einfach ein Testpost in Slack.

<!-- Open Graph — Vorschaukarte für LinkedIn, WhatsApp, Slack, Facebook … -->
<meta property="og:type" content="website" />
<meta property="og:url" content="https://www.lumeo.de/" />
<meta property="og:title" content="Session-Replays und Produkt-Analytics — Lumeo" />
<meta property="og:description" content="Sieh exakt, wie Nutzer dein Produkt bedienen. DSGVO-konform, in 5 Minuten startklar." />

<!-- Bild: 1200 × 630 px (Verhältnis 1,91:1), unter 1 MB, JPG/PNG — absolute URL! -->
<meta property="og:image" content="https://www.lumeo.de/og/start-1200x630.png" />
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />
<meta property="og:image:alt" content="Lumeo-Dashboard mit Session-Replay und Funnel-Ansicht" />
<meta property="og:locale" content="de_DE" />
<meta property="og:site_name" content="Lumeo" />

<!-- Twitter/X — nutzt og:* als Fallback, der Karten-Typ ist aber Pflicht -->
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content="Session-Replays und Produkt-Analytics — Lumeo" />
<meta name="twitter:description" content="Sieh exakt, wie Nutzer dein Produkt bedienen. DSGVO-konform, in 5 Minuten startklar." />
<meta name="twitter:image" content="https://www.lumeo.de/og/start-1200x630.png" />

SEO 3 · JSON-LD Organization & WebSite @graph data-theme="dark"

03

Organization & WebSite

Dieses JSON-LD sagt Suchmaschinen maschinenlesbar, wer hinter der Seite steckt — Grundlage für das Logo im Knowledge Panel und eine saubere Markensuche. Es gehört einmalig in den <head> der Startseite; die @id-Verweise verknüpfen beide Objekte, sodass die WebSite auf die Organisation zeigt.

In sameAs nur Profile eintragen, die es wirklich gibt — leere oder tote Links schaden mehr, als sie nützen.

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@graph": [
    {
      "@type": "Organization",
      "@id": "https://www.lumeo.de/#organization",
      "name": "Lumeo GmbH",
      "url": "https://www.lumeo.de/",
      "logo": "https://www.lumeo.de/logo-512x512.png",
      "sameAs": [
        "https://www.linkedin.com/company/lumeo",
        "https://github.com/lumeo"
      ],
      "contactPoint": {
        "@type": "ContactPoint",
        "contactType": "customer support",
        "email": "support@lumeo.de",
        "availableLanguage": ["de", "en"]
      }
    },
    {
      "@type": "WebSite",
      "@id": "https://www.lumeo.de/#website",
      "url": "https://www.lumeo.de/",
      "name": "Lumeo",
      "inLanguage": "de",
      "publisher": { "@id": "https://www.lumeo.de/#organization" }
    }
  ]
}
</script>

SEO 4 · JSON-LD FAQPage passt zu faq.html data-theme="dark"

04

FAQPage

Macht dein FAQ für Suchmaschinen auswertbar. Die beiden Beispiel-Fragen entsprechen wörtlich dem Baustein FAQ 1 in faq.html — genau so muss es sein: Fragen und Antworten im JSON-LD müssen 1:1 mit dem sichtbaren FAQ auf der Seite übereinstimmen. Unsichtbare Zusatzfragen wertet Google als Spam.

Für jede weitere sichtbare Frage einfach ein zusätzliches Question-Objekt ins mainEntity-Array hängen.

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Kann ich jederzeit kündigen?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Ja. Monats-Abos sind zum Ende des laufenden Abrechnungsmonats kündbar, Jahres-Abos zum Ende des Vertragsjahres — direkt in den Einstellungen, ohne Anruf und ohne Frist von mehreren Monaten. Nach der Kündigung wechselt dein Konto automatisch in den kostenlosen Starter-Plan."
      }
    },
    {
      "@type": "Question",
      "name": "Ist das Produkt DSGVO-konform?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Ja. Alle Daten liegen in einem Rechenzentrum in Frankfurt am Main, IP-Adressen werden anonymisiert und sensible Eingabefelder standardmäßig maskiert. Einen Auftragsverarbeitungsvertrag (AVV) kannst du dir mit einem Klick im Dashboard generieren."
      }
    }
  ]
}
</script>

SEO 5 · JSON-LD SoftwareApplication & Offers passt zu preise.html data-theme="dark"

05

SoftwareApplication mit Preisen

Für SaaS-Produkte ist SoftwareApplication mit offers der passende Typ — die drei Angebote entsprechen den Preiskarten in preise.html: Starter 0 €, Pro 29 €, Business 79 € (jeweils Monatspreis bei monatlicher Zahlung). Wie beim FAQ gilt: Die Preise im JSON-LD müssen mit den sichtbaren Preisen übereinstimmen.

Ein aggregateRating nur ergänzen, wenn echte, auf der Seite sichtbare Bewertungen existieren — sonst weglassen.

SEO 6 · JSON-LD BreadcrumbList passt zu utility.html

06

BreadcrumbList

Zeigt in den Suchergebnissen den Klickpfad statt der nackten URL. Das Beispiel entspricht dem Breadcrumbs-Baustein Utility 5 in utility.html: Die Reihenfolge steckt in position, die letzte Ebene (die aktuelle Seite) bekommt kein item. Das Snippet gehört in den <head> der jeweiligen Unterseite.

Sichtbare Brotkrumen und JSON-LD müssen dieselben Ebenen in derselben Reihenfolge zeigen.

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "Startseite", "item": "https://www.lumeo.de/" },
    { "@type": "ListItem", "position": 2, "name": "Blog", "item": "https://www.lumeo.de/blog/" },
    { "@type": "ListItem", "position": 3, "name": "Produkt-Analytics", "item": "https://www.lumeo.de/blog/produkt-analytics/" },
    { "@type": "ListItem", "position": 4, "name": "Session-Replays DSGVO-konform einsetzen" }
  ]
}
</script>

SEO 7 · hreflang DE/EN x-default

07

hreflang für DE/EN

Gibt es die Seite auf Deutsch und Englisch, sagt hreflang Google, welche Sprachversion wem gezeigt wird — und verhindert, dass beide als Duplikate konkurrieren. Entscheidend: Jede Sprachversion trägt das komplette Set inklusive Verweis auf sich selbst; einseitige Verweise werden ignoriert. x-default fängt alle übrigen Sprachen ab.

Voraussetzung ist eine parallele Seitenstruktur — jede DE-Seite hat ein EN-Gegenstück, z. B. /preise/ ↔ /en/pricing/. Passt zum Sprach-Umschalter in utility.html.

<!-- Identisch in den <head> BEIDER Sprachversionen — jede nennt alle, auch sich selbst -->
<link rel="alternate" hreflang="de" href="https://www.lumeo.de/preise/" />
<link rel="alternate" hreflang="en" href="https://www.lumeo.de/en/pricing/" />

<!-- x-default: Fallback für alle übrigen Sprachen (hier: die deutsche Hauptseite) -->
<link rel="alternate" hreflang="x-default" href="https://www.lumeo.de/preise/" />

SEO 8 · robots.txt & sitemap.xml im Domain-Root data-theme="dark"

08

robots.txt & sitemap.xml

Für eine einzelne, gut verlinkte Landing Page sind beide Dateien optional — Google findet sie auch so. Sobald mehrere Unterseiten existieren (Blog, Docs, Rechtsseiten) oder Bereiche wie /danke/ nicht gecrawlt werden sollen, lohnen sich beide: robots.txt steuert das Crawlen, sitemap.xml beschleunigt das Entdecken neuer Seiten. Beide liegen immer im Root der Domain.

Achtung: Disallow blockiert nur das Crawlen. Soll eine Seite sicher aus dem Index bleiben (z. B. die Danke-Seite), gehört zusätzlich <meta name="robots" content="noindex"> in deren <head>.

robots.txt

# robots.txt — liegt unter https://www.lumeo.de/robots.txt
User-agent: *
Allow: /

# Bereiche, die kein Crawler besuchen soll
Disallow: /danke/
Disallow: /intern/

# Sitemap immer mit absoluter URL angeben
Sitemap: https://www.lumeo.de/sitemap.xml

sitemap.xml

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://www.lumeo.de/</loc>
    <lastmod>2026-06-02</lastmod>
  </url>
  <url>
    <loc>https://www.lumeo.de/preise/</loc>
    <lastmod>2026-06-02</lastmod>
  </url>
  <url>
    <loc>https://www.lumeo.de/en/</loc>
    <lastmod>2026-06-02</lastmod>
  </url>
</urlset>

Abschluss · SEO-Checkliste 10 Punkte vor dem Livegang

Checkliste

10 Punkte vor dem Livegang

Jeder Punkt kostet höchstens ein paar Minuten — spart aber Rankings, kaputte Share-Vorschauen und doppelten Content.

  • Title: 50–60 Zeichen, pro Seite einzigartigKeyword vorn, Marke hinten — nie zweimal derselbe Title.
  • Description: 120–155 Zeichen mit NutzenKonkretes Versprechen + Handlungsaufforderung, keine Keyword-Liste.
  • Genau eine <h1> pro SeiteDanach lückenlose Hierarchie: h2 → h3, keine Ebene überspringen.
  • Canonical gesetzt, eine URL-Variantehttps und www/non-www konsequent, alle anderen Varianten per 301.
  • OG-Bild 1200 × 630 px vorhandenShare-Vorschau mit LinkedIn Post Inspector oder Slack-Testpost geprüft.
  • JSON-LD validiertGoogle Rich-Results-Test und Schema-Validator laufen ohne Fehler durch.
  • Strukturierte Daten = sichtbarer InhaltFAQ-Fragen und Preise im JSON-LD stimmen 1:1 mit der Seite überein.
  • hreflang wechselseitig komplettJede Sprachversion nennt alle Versionen inklusive sich selbst + x-default.
  • robots.txt & Sitemap erreichbarBeide im Domain-Root, Sitemap in der Search Console eingereicht.
  • Alt-Texte & Mobil-CheckJedes inhaltliche Bild beschrieben, Seite ab 360 px Breite fehlerfrei nutzbar.
